01

Apa Itu Spring Boot — dan Kenapa Ada?

Sebelum kita menulis satu baris kode pun, kamu harus tahu masalah apa yang Spring Boot selesaikan. Kalau tidak tahu masalahnya, kamu tidak akan menghargai solusinya.

🎯 Analogi

Bayangkan kamu mau bikin kopi. Spring (framework asli) adalah seperti membeli biji kopi mentah, roaster sendiri, grinder, portafilter, mesin espresso manual — semuanya perlu di-setup dari nol, disambungkan satu per satu, dikonfigurasi tekanan dan suhunya. Spring Boot adalah seperti coffee maker otomatis: kamu tinggal masukkan kopi, tekan tombol, dan kopi jadi. Di balik layar prosesnya sama, tapi setup-nya jauh lebih cepat.

Sejarah Singkat

Spring Framework lahir tahun 2002 sebagai alternatif dari EJB (Enterprise JavaBeans) yang sangat kompleks. Spring lebih simpel, tapi seiring waktu, konfigurasinya sendiri menjadi kompleks. Kamu perlu menulis banyak XML, mendaftarkan setiap bean secara manual, dll.

Tahun 2014, Spring Boot 1.0 diluncurkan dengan filosofi: "Convention over Configuration" — ikuti konvensi standar, dan framework akan otomatis mengkonfigurasi dirinya sendiri.

Tiga Filosofi Utama Spring Boot

1. Opinionated Defaults — Spring Boot punya "pendapat" tentang cara terbaik setup sebuah app. Kamu bisa override, tapi defaultnya sudah masuk akal.

2. Convention over Configuration — Ikuti konvensi penamaan dan struktur folder, maka kamu tidak perlu config apapun secara manual.

3. Production-Ready Out of the Box — Metrics, health checks, logging — semua sudah tersedia tanpa setup tambahan.

02

Spring vs Spring Boot — Perbedaan Nyata

Ini yang sering bikin bingung pemula. Spring Boot bukan pengganti Spring — ia adalah layer di atas Spring yang menghilangkan boilerplate.

Aspek Spring Framework Spring Boot
Konfigurasi Web Server Manual — deploy ke Tomcat eksternal, butuh web.xml Embedded Tomcat otomatis — langsung run jar
Dependency Management Kamu tentukan sendiri versi tiap library, rawan konflik BOM (Bill of Materials) — versi sudah dikelola, tidak konflik
Konfigurasi Bean XML panjang atau @Configuration manual Auto-configuration berdasarkan classpath
Cara Jalankan Perlu application server (Tomcat, JBoss, dll) java -jar app.jar (self-contained)
main() method Tidak ada — di-deploy sebagai WAR Ada! SpringApplication.run() sebagai entry point
Monitoring Perlu setup manual Spring Actuator siap pakai
⚠ PENTING Spring Boot masih menggunakan Spring di dalamnya. Semua konsep Spring (IoC, DI, AOP) tetap berlaku. Spring Boot hanya menghilangkan konfigurasi yang membosankan.

Perbandingan Kode Nyata

Berikut contoh setup Web MVC di Spring biasa vs Spring Boot:

Spring Framework (manual) — WebConfig.java JAVA
// ❌ Spring biasa — kamu harus tulis semua ini sendiri @Configuration @EnableWebMvc public class WebConfig implements WebMvcConfigurer { @Bean public InternalResourceViewResolver viewResolver() { InternalResourceViewResolver resolver = new InternalResourceViewResolver(); resolver.setPrefix("/WEB-INF/views/"); resolver.setSuffix(".jsp"); return resolver; } @Bean public DispatcherServlet dispatcherServlet() { ... } @Bean public ServletRegistrationBean servletRegistration() { ... } // ... masih banyak lagi }
Spring Boot — HelloApplication.java JAVA
// ✅ Spring Boot — ini CUKUP untuk web app yang berjalan @SpringBootApplication public class HelloApplication { public static void main(String[] args) { SpringApplication.run(HelloApplication.class, args); } }

Keduanya melakukan hal yang sama — tapi Spring Boot mengurus semua konfigurasi di balik layar.

Auto-Configuration — Bedah Tuntas

Ini adalah jantung Spring Boot. Kalau kamu paham ini, tidak ada yang "magic" lagi.

Cara Kerja Auto-Configuration

App Start
→
Scan Classpath
→
Cek @Conditional
→
Register Beans
→
App Ready

Spring Boot membaca file khusus bernama spring.factories (Spring Boot 2.x) atau AutoConfiguration.imports (Spring Boot 3.x) yang berisi daftar semua class auto-configuration.

spring-boot-autoconfigure-3.x.jar → META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports TEXT
# Hanya sebagian kecil dari ratusan class yang ada org.springframework.boot.autoconfigure.web.servlet.WebMvcAutoConfiguration org.springframework.boot.autoconfigure.data.jpa.JpaRepositoriesAutoConfiguration org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration org.springframework.boot.autoconfigure.thymeleaf.ThymeleafAutoConfiguration

Spring Boot akan mencoba menjalankan setiap class itu — tapi class itu dilindungi oleh @Conditional annotations. Artinya, sebuah konfigurasi hanya akan aktif jika kondisi tertentu terpenuhi.

@Conditional Annotations — Kunci Auto-Configuration

Mari kita lihat bagaimana DataSourceAutoConfiguration bekerja secara nyata:

DataSourceAutoConfiguration.java (disederhanakan) JAVA
@Configuration @ConditionalOnClass(DataSource.class) // ① Hanya aktif jika DataSource ada di classpath @ConditionalOnMissingBean(DataSource.class) // ② Hanya aktif jika kamu BELUM buat DataSource sendiri @EnableConfigurationProperties(DataSourceProperties.class) public class DataSourceAutoConfiguration { @Bean public DataSource dataSource(DataSourceProperties properties) { // Otomatis baca dari application.properties: // spring.datasource.url, username, password return properties.initializeDataSourceBuilder().build(); } }

Daftar @Conditional yang Sering Dipakai

AnnotationArtinya
@ConditionalOnClassAktif jika class tertentu ada di classpath (ada dependencynya)
@ConditionalOnMissingBeanAktif jika bean tertentu BELUM didaftarkan (kamu bisa override!)
@ConditionalOnPropertyAktif jika property tertentu ada di application.properties
@ConditionalOnWebApplicationAktif hanya di web application
@ConditionalOnBeanAktif jika bean tertentu SUDAH ada
🔍 Insight Penting

@ConditionalOnMissingBean inilah yang memungkinkan kamu override auto-configuration. Kalau kamu mendaftarkan bean sendiri (misalnya DataSource kustom), maka Spring Boot tidak akan membuat bean auto-config-nya. Kamu selalu punya kendali penuh.

Melihat Auto-Configuration yang Aktif

Tambahkan ini ke application.properties untuk melihat apa yang aktif:

src/main/resources/application.properties PROPERTIES
# Lihat semua auto-configuration yang aktif dan tidak aktif logging.level.org.springframework.boot.autoconfigure=DEBUG # Atau jalankan dengan flag ini: # java -jar app.jar --debug

Kamu akan melihat output seperti ini di console:

Console Output (debug mode) LOG
============================ CONDITIONS EVALUATION REPORT ============================ Positive matches: ← AUTO-CONFIG YANG AKTIF DataSourceAutoConfiguration matched: - @ConditionalOnClass found required classes 'DataSource' (OnClassCondition) Negative matches: ← YANG TIDAK AKTIF (dan kenapa) MongoAutoConfiguration: - @ConditionalOnClass did not find required class 'MongoClient' (OnClassCondition)

04

Starter Projects — Dependensi Yang Dikurasi

Sebelum Spring Boot, kalau mau buat web app dengan JPA, kamu harus manually menambahkan: spring-webmvc, spring-orm, hibernate-core, tomcat-embed, jackson-databind, slf4j, logback, dan banyak lagi — dengan versi yang kompatibel satu sama lain. Sangat menyita waktu.

Starter project adalah POM/dependency yang sudah mengemas semua dependensi yang dibutuhkan untuk satu use case tertentu, dengan versi yang sudah diuji kompatibilitasnya.

Starter yang Paling Sering Dipakai

pom.xml — Starter Examples XML
<!-- Untuk Web / REST API --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> </dependency> <!-- ↑ Otomatis include: spring-webmvc, tomcat-embed, jackson-databind, spring-core, spring-context --> <!-- Untuk Database / JPA --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-data-jpa</artifactId> </dependency> <!-- ↑ Otomatis include: hibernate-core, spring-data-jpa, spring-jdbc, transaction management --> <!-- Untuk Testing --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-test</artifactId> <scope>test</scope> </dependency> <!-- ↑ Otomatis include: JUnit 5, Mockito, AssertJ, Spring Test -->

Bagaimana Versi Dikelola — Spring Boot BOM

Di bagian <parent> pom.xml kamu ada ini:

pom.xml — Parent POM XML
<parent> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-parent</artifactId> <version>3.2.0</version> <!-- satu versi untuk semuanya --> </parent> <!-- Karena ada parent di atas, kamu TIDAK perlu tulis versi untuk semua spring dependencies: --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> <!-- tidak perlu <version> -- diwarisi dari parent --> </dependency>
ℹ️ INFO spring-boot-starter-parent mewarisi dari spring-boot-dependencies yang berisi BOM (Bill of Materials) — sebuah daftar ratusan library dengan versi yang sudah diuji kompatibilitasnya. Ini yang memastikan tidak ada dependency conflict.

Spring Initializr — Project Scaffolding

start.spring.io adalah web app resmi untuk generate kerangka proyek Spring Boot. Ini menghemat waktu setup awal secara signifikan.

Cara Pakai Spring Initializr

  1. Buka start.spring.io di browser kamu
  2. Pilih Project: Maven atau Gradle (rekomendasinya Maven untuk pemula)
  3. Pilih Language: Java
  4. Pilih Spring Boot version: Pilih yang stable (bukan SNAPSHOT)
  5. Isi Project Metadata: Group (com.belajar), Artifact (hello-app), dll
  6. Pilih Dependencies: Klik "ADD DEPENDENCIES" → cari "Spring Web"
  7. Klik GENERATE → download ZIP → extract → buka di IDE

Alternatif: Via IntelliJ IDEA

File → New → Project → Spring Initializr (sudah built-in di IntelliJ Ultimate)

Alternatif: Via CLI (Spring Boot CLI)

Terminal BASH
# Install Spring Boot CLI dulu spring init --dependencies=web --name=hello-app hello-app # Atau dengan curl langsung dari start.spring.io API curl https://start.spring.io/starter.zip \ -d dependencies=web,actuator \ -d name=hello-app \ -d packageName=com.belajar.hello \ -o hello-app.zip

Struktur Proyek — Setiap File Punya Makna

Setelah kamu generate proyek, kamu akan melihat struktur ini. Mari kita bedah setiap bagiannya:

Project Structure
hello-app/ ├── src/ │ ├── main/ │ │ ├── java/ │ │ │ └── com/belajar/hello/ │ │ │ └── HelloApplication.java ← Entry point aplikasi │ │ └── resources/ │ │ ├── application.properties ← Semua konfigurasi di sini │ │ ├── static/ ← CSS, JS, gambar (served langsung) │ │ └── templates/ ← Template HTML (Thymeleaf, dll) │ └── test/ │ └── java/ │ └── com/belajar/hello/ │ └── HelloApplicationTests.java ← Test class ├── mvnw / mvnw.cmd ← Maven Wrapper (tidak perlu install Maven) ├── .mvn/ ← Config Maven Wrapper └── pom.xml ← Dependency & build config

File-File Penting — Dibedah Satu Per Satu

HelloApplication.java JAVA
package com.belajar.hello; import org.springframework.boot.SpringApplication; import org.springframework.boot.autoconfigure.SpringBootApplication; @SpringBootApplication // ← Ini satu annotation yang menggabungkan 3 hal: public class HelloApplication { public static void main(String[] args) { SpringApplication.run(HelloApplication.class, args); // SpringApplication.run() melakukan: // 1. Membuat ApplicationContext // 2. Menjalankan semua auto-configuration // 3. Start embedded web server (Tomcat) } }

@SpringBootApplication adalah shortcut untuk 3 annotation:

  • @Configuration — class ini sebagai sumber bean definition
  • @EnableAutoConfiguration — aktifkan auto-configuration
  • @ComponentScan — scan semua @Component, @Service, @Repository di package ini dan sub-package-nya
pom.xml XML
<?xml version="1.0" encoding="UTF-8"?> <project> <!-- 1. Warisi semua default Spring Boot --> <parent> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-parent</artifactId> <version>3.2.0</version> </parent> <!-- 2. Identitas proyek kamu --> <groupId>com.belajar</groupId> <artifactId>hello-app</artifactId> <version>0.0.1-SNAPSHOT</version> <!-- 3. Java version --> <properties> <java.version>17</java.version> </properties> <!-- 4. Dependency - cukup nama tanpa versi --> <dependencies> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> </dependency> </dependencies> <!-- 5. Plugin untuk build executable JAR --> <build> <plugins> <plugin> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-maven-plugin</artifactId> <!-- Plugin ini yang bikin "fat JAR" -- semua dependencies di-bundle jadi satu file JAR yang bisa dijalankan --> </plugin> </plugins> </build> </project>

Ini adalah file konfigurasi utama kamu. Spring Boot membaca file ini saat startup dan menggunakannya untuk mengkonfigurasi beans-nya.

application.properties PROPERTIES
# Port server (default: 8080) server.port=9090 # Nama aplikasi spring.application.name=hello-app # Database config (kalau pakai JPA) spring.datasource.url=jdbc:mysql://localhost:3306/mydb spring.datasource.username=root spring.datasource.password=secret # Log level logging.level.com.belajar=DEBUG # Custom property (bisa dibaca di kode via @Value) app.message=Halo dari config!

Kamu juga bisa pakai format application.yml — lebih bersih untuk config yang bersarang.

Maven Wrapper memungkinkan kamu menjalankan Maven tanpa harus install Maven secara global. Ini memastikan semua developer di tim menggunakan versi Maven yang sama.

TerminalBASH
# Linux/Mac — pakai mvnw ./mvnw spring-boot:run # Windows — pakai mvnw.cmd mvnw.cmd spring-boot:run # Sama seperti: mvn spring-boot:run # tapi tidak perlu install mvn di system

Build Hello World REST API — Step by Step

Sekarang kita bangun REST API sederhana. Kita akan jelaskan setiap baris kodenya.

Langkah 1: Buat Controller

Controller adalah class yang menangani HTTP request. Dalam Spring MVC, kita pakai @RestController.

src/main/java/com/belajar/hello/HelloController.java JAVA
package com.belajar.hello; import org.springframework.web.bind.annotation.*; import org.springframework.beans.factory.annotation.Value; @RestController // ① = @Controller + @ResponseBody // Setiap method return value langsung jadi HTTP response body @RequestMapping("/api") // ② Base URL untuk semua endpoint di class ini public class HelloController { // Membaca value dari application.properties @Value("${app.message:Halo, Dunia!}") // ③ Default value setelah ':' private String message; // GET /api/hello @GetMapping("/hello") public String hello() { return message; // Spring otomatis serialize ke response body } // GET /api/hello/{name} — path variable @GetMapping("/hello/{name}") public String helloName(@PathVariable String name) { return "Halo, " + name + "!"; } // GET /api/greet?name=Budi — query param @GetMapping("/greet") public String greet(@RequestParam(defaultValue="Teman") String name) { return "Selamat datang, " + name + "!"; } // POST /api/hello — terima JSON body @PostMapping("/hello") public String createHello(@RequestBody HelloRequest request) { return "Diterima: " + request.getMessage(); } }

Langkah 2: Buat Request DTO (Data Transfer Object)

src/main/java/com/belajar/hello/HelloRequest.java JAVA
package com.belajar.hello; // Simple POJO — Jackson (auto-included via starter-web) // akan otomatis convert JSON ↔ Java object public class HelloRequest { private String message; // Getter & Setter wajib ada untuk Jackson serialization public String getMessage() { return message; } public void setMessage(String message) { this.message = message; } } // Atau pakai Java Record (Java 14+) — lebih ringkas: // public record HelloRequest(String message) {}

Langkah 3: Tambahkan Config ke application.properties

src/main/resources/application.properties PROPERTIES
spring.application.name=hello-app server.port=8080 app.message=Halo dari Spring Boot!

Langkah 4: Jalankan Aplikasi

Terminal BASH
# Cara 1: Via Maven Wrapper ./mvnw spring-boot:run # Cara 2: Via IDE — klik kanan HelloApplication.java → Run # Cara 3: Build dulu, lalu run JAR ./mvnw clean package java -jar target/hello-app-0.0.1-SNAPSHOT.jar

📌 Ringkasan Day 1

Spring Boot bukan magic — ia adalah sistem yang cerdas berbasis @Conditional: konfigurasi hanya aktif jika kondisi terpenuhi (class ada di classpath, bean belum ada, dll).

Auto-configuration bekerja lewat ratusan class di AutoConfiguration.imports, masing-masing dilindungi @ConditionalOnClass, @ConditionalOnMissingBean, dll.

Starter Projects adalah kumpulan dependency yang dikurasi dengan versi kompatibel, dikelola lewat Spring Boot BOM di parent POM.

Kamu selalu bisa override auto-configuration: daftarkan bean sendiri, dan Spring Boot akan mundur karena @ConditionalOnMissingBean.
